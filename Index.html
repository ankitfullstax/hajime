Basic Commands

git init: Initialize a new Git repository.

git init my-repo

This command creates a new directory named my-repo (if it doesn't already exist) and initializes a new Git repository in it. If you run git init without specifying a directory, it initializes a Git repository in the current directory.

Learn more about git init

git clone <url>: Clone an existing repository from the specified URL.

git clone https://github.com/user/repo.git

This command copies the repository from the provided URL to your local machine. It creates a new directory named after the repository and initializes it with the content of the remote repository.

Learn more about git clone

git status: Show the working tree status.

git status

This command displays the status of your working directory and the staging area. It shows which changes have been staged, which are in the working directory, and which files aren’t being tracked by Git.

git log: Displays a list of recent commits, including their hashes, authors, dates, and messages.

git log

This command is fundamental for understanding the history of changes in a repository. It provides a chronological list of commits, which includes details such as commit messages, authors, and timestamps.

Learn more about git status

git add <file>: Add file contents to the index (staging area).

git add index.html

This command stages the file index.html for the next commit. It tells Git to include the changes to this file in the next commit. To stage all changed files, you can use git add .

Learn more about git add

git commit -m "message": Record changes to the repository with a message.

git commit -m "Added new feature"

This command creates a new commit with the changes you’ve staged and attaches a descriptive message. The message "Added new feature" describes the changes included in this commit.

Learn more about git commit

Branching

git branch: List, create, or delete branches.

git branch

This command lists all the branches in your repository. The current branch is highlighted with an asterisk (*). To create a new branch, you can use git branch new-feature.

Learn more about git branch

git checkout <branch>: Switch to a different branch.

git checkout develop

This command switches your working directory to the branch named develop. If the branch does not exist, you will receive an error. To create and switch to a new branch, use git checkout -b new-branch.

Learn more about git checkout

git merge <branch>: Merge changes from another branch into the current branch.

git merge feature-branch

This command merges the changes from the branch named feature-branch into the current branch. It combines the commit histories of both branches. Resolve any conflicts that arise during the merge process.

Learn more about git merge

Remote Repositories

git remote add <name> <url>: Add a remote repository.

git remote add origin https://github.com/user/repo.git

This command adds a remote repository named origin with the specified URL. You use this remote name to interact with the remote repository for fetch, push, and pull operations.

Learn more about git remote

git fetch: Download objects and refs from another repository.

git fetch origin

This command fetches updates from the remote repository named origin but does not automatically merge them into your local branches. It updates your local copies of the remote branches.

Learn more about git fetch

git pull: Fetch from a remote repository and merge.

git pull origin main

This command fetches updates from the remote branch main and automatically merges them into your current branch. It is a combination of git fetch and git merge.

Learn more about git pull

git push: Update remote refs along with associated objects.

git push origin main

This command pushes your local commits to the remote repository named origin for the branch main. It updates the remote repository with your changes.

Learn more about git push

Advanced Commands

git rebase: Reapply commits on top of another base tip.

git rebase main

This command reapplies your local commits on top of the main branch. It is often used to incorporate upstream changes into your branch, resulting in a linear commit history.

Learn more about git rebase

git cherry-pick <commit>: Apply the changes introduced by some existing commits.

git cherry-pick a1b2c3d4

This command applies the changes from the commit with hash a1b2c3d4 to your current branch. It is useful for picking specific commits from another branch.

Learn more about git cherry-pick

git stash: Stash the changes in a dirty working directory.

git stash

This command temporarily saves your uncommitted changes and reverts your working directory to match the HEAD commit. To reapply the changes, use git stash pop.

Learn more about git stash
